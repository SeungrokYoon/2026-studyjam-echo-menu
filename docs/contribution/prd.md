# Domain B: Contributor & Gamification Domain PRD

본 문서는 일반 비장애인 사용자의 자발적 메뉴판 등록 기여 및 크롬 공룡 게이미피케이션 동기부여 도메인의 최종 기능 요구사항(PRD) 명세서입니다.

---

## 1. 하이브리드 매장 기여 인터페이스 (Hybrid Venue Contribution)
* **요구사항 1:** 기여 페이지에서 📍 내 위치 버튼 터치 시, 브라우저 Geolocation GPS 정보를 획득하여 내 주변 반경 50m 이내 추천 매장 목록을 드롭다운에 노출해야 한다.
* **요구사항 2:** 기여자가 특정 텍스트("스타벅스 홍대")를 입력 시 Google Maps Autocomplete API가 연동되어 전국 일치 장소 목록을 자동완성 팝업 리스트로 제공해야 한다.
* **요구사항 3:** 기여 매장 확정 즉시 미니 지도 캔버스(`#contrib-map-canvas`) 상에 해당 매장의 위경도 기반 마커 핀을 동적 렌더링하고, 고유한 **Google Place ID** 데이터를 획득해야 한다.
* **요구사항 4:** 획득된 Place ID를 Key값으로 하여 업로드한 메뉴판 이미지 및 매장 이름 데이터를 Firestore NoSQL DB에 업서트(Upsert) 해야 한다.
* **요구사항 5:** 한 번의 기여에서 메뉴판 사진을 최소 1장, 최대 10장까지 업로드할 수 있어야 하며, 클라이언트와 서버가 동일한 장수·형식·용량 제한을 검증해야 한다.

---

## 2. 크롬 디노(Dino) 100단계 부화-진화 게이미피케이션 (Dino Gamification)
* **요구사항 1:** 기여자가 신규 매장 개척 시 150pt, 기존 메뉴 업데이트 시 30pt를 적립한다.
* **요구사항 2:** 누적 적립 점수에 따라 사용자의 크롬 픽셀 공룡 프로필이 알(🥚, 1-10Lv) ➔ 아기 공룡(🦕, 11-30Lv) ➔ 달리는 공룡(🦖, 31-60Lv) ➔ 익룡(🦅, 61-90Lv) ➔ 히어로 디노(👑🦖, 91-100Lv)로 100단계 분할 진화 렌더링되어야 한다.
* **요구사항 3:** 랭킹 보드 대시보드 클릭 시, BigQuery 통계를 경유하여 실시간 기여 포인트 순위 상위 100명의 리더보드를 가시화하여 명예의 전당 점수를 조회할 수 있게 한다.
* **요구사항 4 (Google OAuth 및 이메일 식별자 매핑):**
  - 기여자의 고유 랭킹 및 100레벨 Dino 레벨을 기기 파편화 없이 추적/기억하기 위해 **Google OAuth 2.0 (Google Identity Services)** 인증을 도입한다.
  - 로그인 성공 시 발급받은 구글 ID Token(JWT)을 백엔드로 보내 서명을 검증하며, 유출 및 위조를 방지한다.
  - 검증 완료된 사용자의 **구글 이메일 주소(Email)**를 유니크 식별 키로 하여 Firestore DB 상의 `contributors/{email}` 도큐먼트 ID로 사용하며, 점수와 레벨 상태를 영구 저장한다.
  - 개인정보 보호를 위해 프론트 리더보드 노출 시 이메일 정보는 철저히 마스킹(Masking) 및 격리 처리한다.

## 3. 자가 성장형 데이터베이스 (Self-Growing DB Pipeline)
* **요구사항 1:** 만약 시각장애인이 미등록 매장에서 주문을 위해 카메라를 비추고 캡처한 비디오 프레임이 백엔드 분석 결과(OCR)를 통해 신규 메뉴판 구조로 판단될 시, 프론트에서 기여 점수를 소급 적립해 주며 Firestore에 신규 매장 템플릿 정보로 자동 이니셜라이징 적재해야 한다.
* **요구사항 2:** 장애인 사용자의 실제 결제 이용 프레임 스트림 자체가 데이터 경제 순환의 일환으로 Firestore에 피드백 루프로 자동 누적(Auto-upsert) 처리되도록 보정한다.

---

## 4. 디노 게이미피케이션 인공지능 모델 믹스 전략 (AI Model Mix Strategy for Dino)
100단계 Dino 진화 시각화 및 리더보드 운영을 최적화하기 위해, 구글의 모델 티어(Nano, Flash/Bannano, Pro)를 기능적 요건에 맞게 다음과 같이 하이브리드로 정교하게 믹싱하여 배치한다.

### 4-1. **Gemini 2.5/2.0 Flash (Bannano)** — 게이미피케이션 규칙 엔진 및 동적 텍스트 생성
* **역할:** 사용자가 업로드한 메뉴판 이미지의 퀄리티(OCR 가독성, 흔들림 여부, 오타 비율)를 실시간 평가하여 기여 포인트를 차등 산정하고, 100단계 Dino 진화 상태에 맞는 위트 있는 격려 스크립트 및 랭킹 카드 시구(Poem/Description)를 초고속 생성한다.
* **선택 이유:** 실시간 비디오 프레임OCR 스캔과 마찬가지로 대규모 기여자 트래픽의 수용 속도가 관건이다. **TTFT 0.5초 미만의 응답성과 Pro 대비 1/10 수준의 압도적인 토큰 가성비**를 지원하여, 대규모 실시간 게임 보드 상호작용에 가장 이상적인 최적의 가성비/가속 모델이다.

### 4-2. **Imagen 3 Pro (GCP Cloud) / Gemini 1.5 Pro** — 100단계 고품질 픽셀 아트 이미지 에셋 원형 제작
* **역할:** 레트로 8-bit 스타일의 크롬 공룡 캐릭터 100단계 진화형(알에서부터 은하계 히어로 디노까지)의 이미지 스프라이트 원형을 정밀 설계하고, 대량 일괄 생성(Batch Generation)할 때 핵심 디자이너로 활용된다.
* **선택 이유:** 픽셀 아트는 점 하나의 일그러짐, 비대칭, 노이즈 유입만으로도 레트로 게임 특유의 심미성이 크게 무너진다. **텍스트 프롬프트 추종성과 복잡한 그리드 스타일 제어력**이 가장 탁월한 최고 사양의 `Pro` 등급 모델과 `Imagen 3` 이미지 엔진의 강력한 융합이 필수적이다.

### 4-3. **Gemini Nano** — 오프라인 환경 기여 프레임 사전 검증 (Local On-device SDK)
* **역할:** 지하철 역사나 하이퍼마켓 내부 등 이동통신 감도가 저조한 지역에서 기여자가 키오스크 사진을 촬영할 때, 서버로 데이터를 보내기 전 모바일 브라우저 환경에서 로컬 머신 비전(OCR 적합성/화면 포함 유무)을 사전 필터링하고 백업 적재를 조율한다.
* **선택 이유:** 오프라인 혹은 극심한 무선 트래픽 지연 상황에서도 클라우드 API 호출 없이 **온디바이스 단독 오프라인 추론**이 가능하여 사용자의 불필요한 이미지 업로드 재시도를 물리적으로 방지한다.
